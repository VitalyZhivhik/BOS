# BOS (Security Analysis System) - Quick Start Guide

## 🚀 Установка и запуск

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Запуск серверной части (с GUI)

**Windows (PowerShell):**
```powershell
python server/gui.py
```

**Windows (CMD):**
```cmd
python server\gui.py
```

**Linux/Mac:**
```bash
python3 server/gui.py
```

### 3. Запуск клиентской части (с GUI)

**Windows (PowerShell):**
```powershell
python client/gui.py
```

**Windows (CMD):**
```cmd
python client\gui.py
```

**Linux/Mac:**
```bash
python3 client/gui.py
```

## 📦 Создание EXE файлов

### Для сервера:
```cmd
build_server.bat
```

### Для клиента:
```cmd
build_client.bat
```

После сборки executables появятся в папке `dist/`:
- `BOS_Server.exe` - серверная часть
- `BOS_Client.exe` - клиентская часть

## 🎯 Как использовать

### Клиентская часть (атакующий):
1. Введите IP адрес цели
2. Укажите диапазон портов (по умолчанию 1-1000)
3. Выберите тип сканирования
4. Нажмите "Start Scan"
5. После завершения нажмите "Send to Server" для сохранения результатов

### Серверная часть:
1. Нажмите "Start Analysis"
2. Система проанализирует установленное ПО
3. Сопоставит уязвимости с базами CVE/CWE/CAPEC/MITRE
4. Определит реализуемые и нереализуемые атаки
5. Нажмите "Export Report" для сохранения отчёта

## 📊 Форматы отчётов

Система поддерживает экспорт в форматах:
- **JSON** - для дальнейшей обработки
- **HTML** - красивый веб-отчёт
- **TXT** - текстовый формат

## ⚠️ Важные замечания

1. **Права доступа**: Для сканирования портов могут потребоваться права администратора
2. **Брандмауэр**: Отключите брандмауэр или добавьте исключения для сканирования
3. **Законность**: Используйте только на системах, которыми владеете или имеете разрешение на тестирование

## 🔧 Решение проблем

### Ошибка PYTHONPATH (Windows PowerShell)
Не используйте синтаксис Linux. Просто запускайте:
```powershell
python server/gui.py
```

### Ошибка "<IP>" в команде
Замените `<IP>` на реальный IP адрес, например:
```powershell
python client/gui.py --target 192.168.1.100
```

### Модуль не найден
Убедитесь, что все зависимости установлены:
```bash
pip install -r requirements.txt
```
