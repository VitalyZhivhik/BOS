# Security Analysis and Attack Simulation System

## Обзор системы

Данная система состоит из двух частей:

### 1. Серверная часть (Server-side Analyzer)
- Анализирует установленное программное обеспечение на сервере
- Определяет средства обеспечения безопасности
- Анализирует инфраструктуру сервера
- Оценивает механизмы защиты

### 2. Клиентская часть (Attacker Simulator)
- Сканирует порты сервера
- Ищет возможные векторы атаки
- Отправляет список потенциальных атак на сервер

## Функциональность

### Серверная часть
- Автоматическое обнаружение установленного ПО
- Идентификация средств безопасности (фаерволы, антивирусы, SIEM)
- Анализ инфраструктуры (базы данных, веб-серверы, службы)
- Интеграция с базами данных уязвимостей

### Клиентская часть
- Сканирование открытых портов
- Определение запущенных служб
- Симуляция атакующих действий
- Отправка результатов на сервер

### Корреляционный движок
- Сопоставление найденных уязвимостей с базами:
  - **CVE** (Common Vulnerabilities and Exposures)
  - **CWE** (Common Weakness Enumeration)
  - **CAPEC** (Common Attack Pattern Enumeration and Classification)
  - **MITRE ATT&CK** (Adversarial Tactics, Techniques, and Common Knowledge)
- Определение реализуемости атак
- Фильтрация нерелевантных угроз

### Система отчётности
- Отчёт о нереализуемых атаках (например, SQL-инъекции при отсутствии БД)
- Рекомендации по устранению уязвимостей
- Приоритизация мер защиты

## Требования

- Python 3.8+
- PostgreSQL или SQLite для хранения данных
- Библиотеки:
  - `scapy` - для сетевого сканирования
  - `requests` - для API запросов
  - `sqlalchemy` - для работы с БД
  - `jinja2` - для генерации отчётов

## Установка

```bash
pip install -r requirements.txt
```

## Использование

### Запуск серверной части
```bash
python server/main.py
```

### Запуск клиентской части
```bash
python client/main.py --target <IP_адрес_сервера>
```

## Пример работы

1. **Сканирование**: Клиент сканирует сервер, находит открытые порты 80, 443, 22
2. **Анализ**: Сервер определяет, что установлены nginx, SSH, но нет СУБД
3. **Корреляция**: Система находит уязвимость CVE-202X-XXXX для SQL-инъекций
4. **Фильтрация**: Атака помечается как "нереализуемая" (нет базы данных)
5. **Отчёт**: Генерируется рекомендация обновить nginx и настроить фаервол
