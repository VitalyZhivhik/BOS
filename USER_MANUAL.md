# Руководство пользователя BOS Client

## О программе

**BOS Client** — это приложение для сканирования сетевых портов и выявления возможных векторов атак. Программа предназначена для **учебных целей** и помогает понять принципы работы сетевой безопасности.

### Возможности программы

- ✅ Сканирование TCP/UDP портов
- ✅ Определение запущенных служб
- ✅ Идентификация векторов атак по базе MITRE ATT&CK
- ✅ Экспорт результатов в различные форматы (JSON, TXT)
- ✅ Интеграция с утилитой ScanOval для поиска уязвимостей
- ✅ Красивый интерфейс на PyQt6 с тёмной темой
- ✅ Полностью русский интерфейс

## Пошаговая инструкция

### 1. Настройка сканирования

1. Введите **IP-адрес** или **имя хоста** цели в соответствующее поле
   - Примеры: `192.168.1.1`, `localhost`, `example.com`

2. Настройте **диапазон портов**:
   - Для быстрого сканирования: `1-1000`
   - Для полного сканирования: `1-65535`
   - Для проверки конкретных портов: `80-443`

3. Выберите **тип сканирования**:
   - **TCP Connect** — полное соединение (рекомендуется для большинства случаев)
   - **SYN Scan** — скрытное сканирование
   - **UDP Scan** — для UDP-сервисов
   - **Service Detection** — с определением версий служб

### 2. Запуск сканирования

Нажмите кнопку **🚀 Начать сканирование**

Во время сканирования:
- Наблюдайте за прогрессом на панели прогресса
- Следите за статистикой в реальном времени
- Читайте журнал активности

### 3. Остановка сканирования

При необходимости нажмите **⏹️ Остановить**. Подтвердите остановку в диалоговом окне.

### 4. Анализ результатов

После завершения сканирования изучите результаты во вкладках:

#### Вкладка "Открытые порты"
- Список всех открытых портов
- Информация о протоколе (TCP/UDP)
- Название службы (если определена)
- Фильтрация по протоколу

#### Вкладка "Векторы атак"
- Выявленные векторы атак
- Ссылки на технику MITRE ATT&CK
- Идентификаторы CAPEC
- Описание каждой угрозы

#### Вкладка "Службы"
- Обнаруженные сетевые службы
- Номера портов
- Версии служб (если определены)

### 5. Экспорт результатов

1. Нажмите **💾 Экспорт результатов**
2. Выберите формат файла:
   - **JSON** — для дальнейшей обработки
   - **TXT** — для чтения человеком
3. Укажите имя файла и место сохранения
4. Нажмите "Сохранить"

### 6. Отправка на сервер

1. Нажмите **📤 Отправить на сервер**
2. Сохраните файл с векторами атак
3. Передайте файл администратору сервера для анализа

## Утилита ScanOval

**ScanOval** — встроенная утилита для поиска информации об уязвимостях по базе OVAL.

### Как использовать ScanOval

1. Перейдите на вкладку **"ScanOval"**
2. Введите **CVE-идентификатор** уязвимости
   - Пример: `CVE-2021-44228` (Log4Shell)
3. Нажмите **🔍 Проверить уязвимость**
4. Изучите полученную информацию

### Примеры известных уязвимостей для проверки

- `CVE-2021-44228` — Log4Shell (Apache Log4j)
- `CVE-2017-0144` — EternalBlue (SMB)
- `CVE-2019-11043` — PHP-FPM Remote Code Execution
- `CVE-2020-1472` — Zerologon (Netlogon)

> **Примечание:** Для полноценной работы ScanOval требуется подключение к серверу с базой OVAL или локальная база уязвимостей.

## Типы сканирования

### TCP Connect
- **Описание:** Полное TCP-соединение (three-way handshake)
- **Преимущества:** Наиболее надёжный метод
- **Недостатки:** Легко обнаруживается системами защиты
- **Когда использовать:** Для легального тестирования в своей сети

### SYN Scan
- **Описание:** Половинчатое соединение (отправляется только SYN-пакет)
- **Преимущества:** Более скрытный метод
- **Недостатки:** Требует специальных привилегий
- **Когда использовать:** Для тестирования скрытности

### UDP Scan
- **Описание:** Сканирование UDP-портов
- **Преимущества:** Обнаружение UDP-сервисов
- **Недостатки:** Медленнее TCP, менее надёжно
- **Когда использовать:** Для поиска DNS, DHCP, SNMP сервисов

### Service Detection
- **Описание:** Определение версий служб
- **Преимущества:** Детальная информация о сервисах
- **Недостатки:** Увеличивает время сканирования
- **Когда использовать:** Для полного аудита безопасности

## Часто задаваемые вопросы (FAQ)

### ❓ Почему сканирование занимает много времени?

**Ответ:** Время сканирования зависит от:
- Количества сканируемых портов
- Скорости сетевого соединения
- Выбранного типа сканирования
- Задержек сети (ping)

**Совет:** Для быстрого тестирования используйте диапазон `1-1000` вместо `1-65535`.

### ❓ Что значит статус "Filtered"?

**Ответ:** Порт блокируется фаерволом или системой фильтрации пакетов. Ответ от порта не получен в течение таймаута.

### ❓ Можно ли сканировать удалённые серверы?

**Ответ:** Только при наличии **письменного разрешения** владельца сервера. Несанкционированное сканирование является нарушением закона.

### ❓ Где сохраняются результаты?

**Ответ:** Результаты сохраняются в выбранную вами папку при экспорте. По умолчанию предлагается имя файла с текущей датой и временем.

### ❓ Как обновить базу уязвимостей ScanOval?

**Ответ:** В текущей версии демонстрируется интерфейс утилиты. Для полноценной работы требуется настройка подключения к API базы OVAL.

### ❓ Программа не запускается. Что делать?

**Ответ:**
1. Убедитесь, что установлен Python 3.8+
2. Проверьте установку зависимостей: `pip install -r requirements.txt`
3. Убедитесь, что установлена PyQt6: `pip install PyQt6`
4. Проверьте наличие графического интерфейса (X11/Wayland на Linux)
